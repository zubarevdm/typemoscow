# Cutover на боевой домен — чеклист

Документ один раз пройти по порядку, когда получите доступ к боевому домену.
Все шаги проверены на staging-окружении.

---

## 0 · Перед началом

- [ ] Убедиться что текущая staging-версия в `main` зелёная (CF Pages → Deployments, последний билд успешен)
- [ ] Сделать локальную копию `astro/src/content/*.json` на всякий случай (бэкап контента)
- [ ] Записать боевой домен в этот документ: `__________.ru` / `.com`

---

## 1 · Подключить домен к Cloudflare Pages

1. Зайти в CF Dashboard → Pages → проект `typemoscowtest`
2. Вкладка **Custom domains** → **Set up a custom domain**
3. Ввести боевой домен (без `https://`, без `www`) → **Continue**
4. CF предложит один из двух сценариев:
   - **Домен уже у Cloudflare** — добавит DNS-запись автоматически за 30 сек
   - **Домен у другого регистратора** — покажет CNAME-запись, которую надо добавить в DNS у регистратора:
     - Type: `CNAME`
     - Name: `@` (или `www`, в зависимости от выбранного хоста)
     - Value: `typemoscowtest.pages.dev`
     - TTL: auto или 3600
5. Дождаться `Active` (обычно 1–5 минут после правильно настроенного DNS)
6. SSL-сертификат CF выпустит сам (Let's Encrypt, бесплатно)

Если хотите ещё и `www.домен.ru` — повторить шаги 2–4 для второго хоста.

## 2 · Переименовать CF Pages проект (опционально)

Имя проекта сейчас `typemoscowtest`. Имя нельзя поменять — но и не нужно: оно
видно только в админке CF и в служебном URL `typemoscowtest.pages.dev`,
который останется как mirror. Можно оставить.

Если очень хочется чистого имени — создать новый проект `typemoscow`, заново
выкатить, удалить старый. Это лишний риск ради косметики — **не рекомендуется.**

---

## 3 · Обновить контент через админку

После того как кастомный домен `Active`, открываем `https://домен.ru/admin`:

1. Войти в админку (пароль из `ADMIN_PASSWORD` в CF Pages)
2. Перейти в **«Сайт»** (последний пункт сайдбара)
3. Поменять поле **Канонический URL** с `https://typemoscowtest.pages.dev` на `https://домен.ru`
4. **Снять галочку «Скрыть сайт от поисковиков»**
5. Нажать **«Опубликовать»**

Один коммит, ~1 минута до автодеплоя.

---

## 4 · Обновить файлы, которые админка не покрывает

Контент в JSON правится через CMS, но эти файлы нужно поправить руками
(одним коммитом). Открыть в редакторе:

### `astro/astro.config.mjs`

Поменять `site: 'https://typemoscowtest.pages.dev'` → `site: 'https://домен.ru'`.

### `astro/public/robots.txt`

Сейчас:
```
User-agent: *
Disallow: /
```

Заменить на:
```
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /api/
Sitemap: https://домен.ru/sitemap-index.xml
```

### `astro/public/sitemap-index.xml`

Генерируется автоматически через `@astrojs/sitemap` на основе `site:` из `astro.config.mjs`.
Отдельно править ничего не нужно — после смены `site:` всё пересоберётся.

### Поиск других хардкодов

```bash
grep -rn "typemoscowtest" astro/src astro/public
```

Должно остаться 0 совпадений. Если что-то нашлось — поправить.

## 5 · Закоммитить и пушнуть

```bash
git add astro/astro.config.mjs astro/public/robots.txt astro/public/sitemap-index.xml
git commit -m "Cutover to <домен.ru>"
git push origin main
```

Через ~1 мин CF Pages автоматически передеплоится.

## 7 · Поисковики

### Яндекс Вебмастер

1. `webmaster.yandex.ru` → Добавить сайт → `https://домен.ru`
2. Подтвердить владение (DNS-запись TXT, проще всего)
3. Раздел **«Файлы Sitemap»** → добавить `https://домен.ru/sitemap-index.xml`
4. Раздел **«Переобход»** → отправить главную и `/works/`

### Google Search Console

1. `search.google.com/search-console` → Add property → `https://домен.ru` (Domain-уровень)
2. Подтвердить через DNS TXT
3. **Sitemaps** → добавить `sitemap-index.xml`
4. **URL Inspection** → запросить индексацию для главной

Индексация занимает от нескольких дней до пары недель — нормально.

## 9 · Бэкап / откат

Бэкап автоматический — весь контент в git. Если что-то пойдёт не так:

```bash
git log --oneline -10        # найти последний хороший коммит
git revert <bad-commit-sha>  # создаст revert-коммит
git push origin main
```

CF Pages пересоберёт. На случай критической поломки можно ещё в CF Dashboard
→ Deployments → у любого старого зелёного билда нажать **«Rollback to this deployment»**.
